编程题--租房管理系统

技术分享
  • c
  • 编程

题目

你的好朋友小王最近开了一家房产中介公司，听闻你是程序员，所以找你帮忙写一个租房管理软件。你愉快的答应了。

需求分析

小王的需求如下：

  1. 支持录入房子和录入租客。
  2. 系统自动匹配符合条件的房子和租客。
  3. 可以生成报表来分析收入情况和哪种房子比较好租。

总体设计

经过一天的思考，你觉得这个程序很简单，只需要实现下面的6个API就可以了：

初始化

说明：

  1. 使系统进入初始状态，清空所有数据。

录入房子信息

说明：

  1. 新录入的房子都是空闲状态可以用来出租。

  2. 房子的属性

    | 类型 | 取值 | 说明 |
    |—-|———–|————————|
    | 面积 | [1,100] | 平方 |
    | 户型 | [1,4] | 1.一室一厅 2.二室一厅 3.三室一厅 4三室 |
    | 朝向 | [1,4] | 1.东 2.南 3.西 4.北 |
    | 电梯 | [0,1] | 0.没有电梯 1.有 |
    | 租金 | [100,20000] | 租金，单位元 |

  3. 布尔类型参考如下定义：

寻找租客

  1. 租客一次结清所有钱款。

  2. 每录入一个租客，即刻在系统内查找满足要求的房子：

    户型要求必须满足

    朝向要求必须满足

    电梯要求必须满足

    面积要求必须满足

    面积相等时选户型最大的

    户型相同时选更便宜的

    租金相同时按房子录入顺序从前到后

  3. 如果录入人时候没有符合条件的房子，则本次录入失败。

  4. 租客的属性：

    | 类型 | 取值 | 说明 |
    |—-|————|———————–|
    | 名字 | 字符串，长度[1,20] | 要求全英文，区分大小写，不会重名 |
    | 户型需求 | [1,3] | 1.一室及以上 2.二室及以上 3.三室及以上 |
    | 朝向需求 | [0,1] | 0.不限 1.朝南 |
    | 面积需求 | [1,100] | 至少需要的房子面积 |
    | 电梯 | [0,1] | 0.不限 2.有电梯 |
    | 租房月数 | [1,12] | 只租整月 |
    ​

退房

说明：

  1. 租房和退房同一天完成。
  2. 租房后可当天退房，但是要付一个月的房租，罚的租金算到该房子的总收入里面。
  3. 退房后可以通过重新录入租客的方式继续租房。

查询

说明：

  1. 只统计产生了收入房子的信息。如果所有的房子未产生收入就报错。

  2. 根据户型，朝向，电梯三个维度综合汇总：即这三个字段值相同的所有房子归为一组，合并为一行输出，同一组的总租金需要叠加。

  3. 汇总表中，按照如下顺序从前到后排列：

    1. 总租金
    2. 朝向（南、东、西、北）
    3. 电梯
    4. 户型（从小到大）

退出

说明：

  1. 退出系统。

交互式终端界面

  1. 初始化完毕之后进入系统，系统显示Enter>提示用户输入命令：

  2. 支持的命令有

    1. help 显示帮助
    2. ih 录入房子，后接参数。
    3. it 录入租客，后接参数。
    4. refund 退款，后接参数。
    5. qr 生成报表
    6. quit 退出系统

  3. 参数顺序和要求见总体设计。

其他要求

  1. 在linux下采用C语言完成本题目。
  2. API声明和结构体定义写到rent.h文件中。
  3. API的实现写到rent.c文件中。
  4. 界面写到main.c文件中。
  5. 要求写一个Makefile。

评分标准

  1. 一共有10个测试用例直接调用API，通过一个得10分。
  2. 一共有10个测试用例通过界面来模拟人工输入，通过一个得10分。
  3. 满分200分。
